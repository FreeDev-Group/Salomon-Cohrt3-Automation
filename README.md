# Student Survey App – Automation Testing (Cohort 3)

This repository contains the automated end-to-end (E2E) testing suite for the **Student Survey Application**, built using Cypress.

The purpose of this project is to simulate a real-world QA environment where each team member independently implements test cases while following a shared standard and structure.

---

## Project Overview

* Testing Framework: Cypress
* Test Type: End-to-End (E2E)
* Application Under Test: [Student Survey App](https://student.michaelkentburns.com/)

## Testing Approach

Each team member works **individually** but tests the **same functionalities**.

This allows:

* Code comparison
* Best practice identification
* Continuous improvement

---

## Standards

All test scripts must:

* Follow clear naming conventions
* Be independent and reusable
* Avoid code duplication
* Use fixtures where applicable
* Include meaningful assertions

---

## Reports

Each contributor must provide a test report in:

```
docs/test-reports/
```

Report should include:

* Tested scenarios
* Issues encountered
* Bugs identified
* Suggested improvements

---

## Branch Strategy

* `main` → Stable and validated version
* `develop` → Active development

All tested and approved code is merged into `main`.
